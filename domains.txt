Maven

Role:
Maven is a build automation and project management tool for Java projects.

Purpose:
- Manages project dependencies.
- Compiles source code.
- Runs tests.
- Packages the application into JAR or WAR files.

Used In:
Build phase of the DevOps lifecycle.


Git

Role:
Git is a distributed version control system used to track changes in source code.

Purpose:
- Maintains version history.
- Supports branching and merging.
- Enables team collaboration.
- Helps restore previous versions of code.

Used In:
Code phase of the DevOps lifecycle.


Docker

Role:
Docker is a containerization platform used to package applications with their dependencies.

Purpose:
- Creates lightweight containers.
- Ensures consistent environments.
- Simplifies application deployment.
- Improves portability across systems.

Used In:
Deploy phase of the DevOps lifecycle.


AWS

Role:
Amazon Web Services (AWS) is a cloud computing platform that provides infrastructure and services for hosting applications.

Purpose:
- Hosts applications in the cloud.
- Provides storage, databases, and networking.
- Offers scalable computing resources.
- Supports high availability and reliability.

Used In:
Deploy and Operate phases of the DevOps lifecycle.

Kubernetes

Role:
Kubernetes is a container orchestration platform used to manage containerized applications.

Purpose:
- Automates deployment.
- Scales applications automatically.
- Manages container health.
- Balances application traffic.

Used In:
Operate phase of the DevOps lifecycle.

Datadog

Role:
Datadog is a cloud monitoring and analytics platform.

Purpose:
- Monitors servers and applications.
- Tracks performance metrics.
- Detects issues in real time.
- Generates alerts for system failures.

Used In:
Monitor phase of the DevOps lifecycle.

Jenkins

Role:
Jenkins is an automation server used for Continuous Integration and Continuous Deployment (CI/CD).

Purpose:
- Automates software builds.
- Executes automated tests.
- Deploys applications.
- Integrates code changes continuously.

Used In:
Release phase of the DevOps lifecycle.
